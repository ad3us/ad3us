Find The Biggest Number

Today I started reading the book “Computer Science Distilled” by W. Wilho. This is a very useful book for students who are studying or want to study Computer Science. And after the preface on the first chapter, I came across a flowchart that showed how the Wikipedia editing works. It looked like this:

I was quite surprised when I saw this. It looked like a version control system, and maybe it was. And after that, the next task was in the style of flowcharts, “Finding the maximum value between three variables.”:

1st method

I tried to implement this in JavaScript. And that’s what happened:

let a = 2;
let b = 3;
let c = 4;

if (a > b) {
	if (a > c) {
		console.log(a, "is the biggest number!")
	} else {
		console.log(c, "is the biggest number!")
	}
} 

if (b > a) {
	if (b > c) {
		console.log(b, "is the biggest number!")
	} else {
		console.log(c, "is the biggest number!")
	}
} 

Output:
4 is the biggest number!

Yes, I know. This code is terrible. But most importantly, it works!

Let’s analyze what I did step by step:

First, we assigning values to variables:

let a = 2;
let b = 3;
let c = 4;

Then, in this conditional block:

if (a > b) {
	if (a > c) {
		console.log(a, "is the biggest number!")
	} else {
		console.log(c, "is the biggest number!")
	}
}

  • The condition a > b evaluates to false because 2 (value of a) is not greater than 3 (value of b).

  • Since the outer if block is not executed, the inner conditions (a > c) are skipped entirely.

In the second conditional block:

if (b > a) {
	if (b > c) {
		console.log(b, "is the biggest number!")
	} else {
		console.log(c, "is the biggest number!")
	}
}

  • The condition b > a evaluates to true because 3 (value of b) is greater than 2 (value of a). The inner condition b > c is then evaluated:

  • b > c evaluates to false because 3 (value of b) is not greater than 4 (value of c).

As a result, the else block is executed:

console.log(c, "is the biggest number!");
